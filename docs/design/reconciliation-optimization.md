# 對帳作業優化：師傅視角與撥款操作改善

## 概述

- **類型**：既有功能優化
- **受影響角色**：客服（會計）

---

## 問題定義

### 痛點

會計對帳作業分為兩個階段，但目前系統只有一個視角支撐：

1. **對帳階段**：確認平台收款與訂單資料是否一致，需要逐筆交易的時序檢視——現有交易明細視角適合此階段，只要確保資料抓取正確即可
2. **撥款操作階段**：確認各師傅應撥金額、填入發票號碼、執行加入匯款記錄——需要以師傅為單位的彙整視角，目前缺乏，導致會計需在 Excel 手動加總各師傅的應撥金額

此外，現有介面有兩個操作問題：
- 「預設」/「自選」按鈕語意不清，導致部分操作人員改用編輯彈窗直接修改狀態，匯款紀錄因此不完整且無法追蹤
- `invoice_no` 欄位存在但未被正確使用，公司戶師傅的發票號碼無法有效記錄

### 現況

- 匯款資訊頁面以單筆交易為單位排列，同一師傅的多筆收入分散於不同列
- 每筆交易需個別點擊「預設」（全額）或「自選」（自訂金額）加入匯款記錄
- 師傅本期應撥總金額需會計在 Excel 手動以 SUMIF 加總
- 編輯彈窗開放修改狀態欄位，造成繞過正規流程的捷徑
- 師傅的請款模式（開立發票 / 整筆個人 / 分期個人）目前無系統欄位記錄

### 成功標準

- 會計在系統內即可直接看到每位師傅本期的應撥總金額與請款模式，無需切換至 Excel
- 一位師傅的所有交易可一鍵批次加入匯款記錄
- 公司戶師傅的發票號碼可在師傅視角統一填入，自動寫入該師傅本期所有交易列
- 狀態變更只能透過正規匯款流程觸發，不再出現直接改狀態的誤操作

---

## 設計方向

### 1. 新增師傅視角的切換方式

在現有匯款資訊頁面提供可切換至師傅視角的介面機制（最可能實作為 Tab，確切形式待定）。

師傅視角每位師傅顯示以下資訊：

| 欄位 | 說明 |
|------|------|
| 師傅名 | 識別用 |
| 應撥總金額 | 前端加總本期該師傅所有 `income` 欄位 |
| 請款模式 | 顯示用，從師傅個人資料讀取（見前提條件） |
| 發票號碼 | 可輸入，統一填入後寫入該師傅本期所有交易列 |

師傅視角支撐的操作：
- **批次加入匯款記錄**：一鍵處理該師傅本期所有交易，於後端產生對應多筆匯款紀錄（匯款紀錄頁面不需改動）
- **invoice_no 統一填入**：沿用現有編輯 API，個人戶師傅留空即可

### 2. 封鎖誤用路徑

編輯彈窗的「狀態」欄位改為唯讀，狀態變更只能透過正規匯款流程觸發。原有直接改狀態的使用情境確認為誤操作，不需額外的替代出口。

### 3. 按鈕重新命名

「預設」/「自選」標籤語意不清，需重新命名為更能反映操作意義的文字（TBD）。

---

## 影響範圍

- **匯款資訊頁面**（後台）：新增師傅視角切換、調整「預設/自選」按鈕文字、編輯彈窗狀態欄改為唯讀
- **師傅個人資料**：新增「請款模式」欄位（前提條件，非本次主要開發項目）
- **master_income 資料表**：`invoice_no` 欄位復用，建立明確填入規範；資料結構不變

## 決策記錄

**批次操作產生多筆匯款紀錄（而非合併為一筆）**
→ 不修改後端資料結構，批次操作於前端分別呼叫現有 API；匯款紀錄頁面行為不變
→ 放棄方案：前端加總後以自選金額送出一筆合併記錄（會改變後端紀錄結構）

**請款模式為師傅個人資料層級欄位**
→ 一經設定不接受變更，屬於師傅屬性而非交易屬性，不存入 master_income 每列
→ 本次設計只需讀取顯示，不建立對應操作邏輯

**invoice_no 以師傅＋期別為單位填入**
→ 公司戶師傅每期開一張發票涵蓋所有收入，填入後統一寫入本期所有列；沿用現有編輯 API
