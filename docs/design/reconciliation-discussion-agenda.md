# 對帳與撥款作業優化：討論內容

## 一、現有功能說明&優化方向

### 師傅收入資料建立時機（已修復）

師傅完成訂單收入資料的建立時機已修正：

- **原邏輯**：客戶完成尾款支付時建立
- **現邏輯**：客戶同意驗收時建立

### 後台匯款資訊的三個區塊

| 區塊 | 功能說明 |
|------|---------|
| **前期未匯** | 顯示前期未匯款/部分匯款的收入資料；可下載 Excel (查看工程師修復狀態) |
| **匯款資訊** | 以逐筆收入資料呈現；透過「預設」（全額）/「自選」（自訂金額）按鈕加入匯款紀錄 |
| **匯款紀錄** | 加入後的彙整紀錄，要再次確認才會寫入紀錄，目前未被使用 |

### 現況痛點

- 每次撥款前，會計需整合三份資料（藍新收款、ezpay 發票、平台訂單）在 Excel 人工對帳，每次約花 5 個工作天
- 同一師傅的多筆收入分散各列，應撥總額需手動加總
- 匯款狀態需逐筆操作每筆收入來標示，無法以師傅為單位批次處理
- 師傅請款模式無系統欄位記錄

### 優化方向（待確認）

- 在匯款資訊頁面新增**師傅視角**介面，以師傅為單位彙整應撥金額、請款模式；並可設定是否已收到師傅發票，支援批次操作(已撥款、未撥款、部分撥款)。
- 「加入匯款記錄 → 確認匯款才寫入」的兩段式流程不直觀，導致確認步驟幾乎未被使用——討論是否翻掉重做（程式邏輯大改），或維持兩段式但加強 UI 引導。

- [ ] **小費發票欄位缺失**：目前下載的 Excel 沒有小費發票號碼欄位，是否需要補上？小費收入目前如何核對？
- [ ] **小費毛利率計算錯誤**：小費收入列的毛利率計算目前抓取母訂單的客收金額作為營收基礎，導致數字失真。
- [ ] **毛利率算法一致性**：匯款資訊介面的毛利率算法與報價單算法有出入，以及匯款資訊介面是否有必要顯示毛利率？
