# Pro 升級頁面：Tiger Points 折抵規則說明與邏輯修正

## 概述

- **類型**：既有功能優化
- **受影響角色**：客戶

---

## 問題定義

### 背景

平台原始政策明確規定，Tiger Points 折抵自升級後的**下一張訂單**起生效，進行中的訂單不適用。此設計呼應平台「超前部署」的行銷訴求——鼓勵客戶在有需求前主動升級。

### 現況

目前系統在付尾款時僅判斷該用戶是否為 Pro 會員，未考量升級時間點與訂單進度的關係。這導致客戶可在收到報價單後才升級，仍能於當前訂單使用點數折抵尾款。

### 客戶行為落差

部分客戶在媒合過程中聽到客服推廣 Pro 方案後，選擇等到看見報價單、確認金額後才決定是否升級。此行為削弱了「提前升級」的誘因，與平台超前部署的行銷訴求不符，也使系統行為偏離原始政策意圖。

### 成功標準

客戶在升級前即理解折抵規則，系統行為回歸原始政策，提前升級的決策能體現應有的價值。

---

## 設計方向

### 後端邏輯修正

新增升級時間點與訂單報價進度的判斷邏輯：

- 升級時訂單**尚未報價** → 可於該訂單尾款使用點數
- 升級時訂單**已完成報價** → 該訂單不適用，點數從下一張訂單起生效

### UI 說明優化

在 Pro 升級頁面兩處新增說明文字，不調整頁面版面結構：

**位置一：Tiger Points Benefit Card 小字**

於現有副標下方新增一行說明，字級縮小一號（建議 11–12px）、灰色、＊ 前綴：

> ＊折抵適用於尚未報價與升級後的新訂單，既有已報價訂單不適用

**位置二：FAQ「TIGER POINTS 可以做什麼用？」**

原有文案後新增第二段：

> TIGER POINTS 為平台贈與之優惠點數，可以折抵本平台修繕訂單尾款或商城商品消費。
>
> 點數升級後立即生效，適用於尚未報價的訂單，以及升級後建立的所有新訂單。既有已報價訂單不適用，請於下一張訂單起使用。

### 文案策略

說明文字採**正面敘述**方式呈現——先說明點數的適用範圍，再帶出例外情況，而非以限制或警告語氣開頭。目的是在銷售頁面維持正向氛圍，同時確保規則清楚傳達。

## 決策記錄

**折抵適用範圍（Policy A）**
決定：師傅報價前升級可折抵當前訂單；報價後升級不適用，從下一張訂單起折抵。
為什麼：與原始政策精神對齊，同時保留訂單進行中可隨時升級的彈性。
放棄的替代方案：
- 進行中訂單一律不適用（更嚴格，需調整現有邏輯）
- 升級後立即可用於任何訂單（最佳即時體驗，但稀釋提前升級誘因）

**視覺處理**
決定：灰色小字 ＋ ＊ 前綴，不使用紅字。
為什麼：定位為補充說明而非警告，維持頁面銷售氛圍。
