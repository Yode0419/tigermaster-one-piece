# Tiger Points 說明頁文案補充與支付頁優化

## 概述

- **類型**：既有功能優化
- **受影響角色**：客戶、客服

---

## 問題定義

### 痛點

每天都有 Pro 會員客戶在支付尾款後，才向客服反映沒有看到點數折抵入口，要求補折抵。客服無法線上處理，須退刷原付款、另開發票做折讓，人工作業繁瑣，對客服造成每日重複性負擔。

### 現況

支付尾款頁面已有「TIGER POINTS 折抵」區塊供客戶輸入折抵點數，旁邊有「說明」按鈕可開啟 Tiger Points bottom sheet。系統功能本身沒有問題。

根本原因是 UX 問題：付款按鈕為置底懸浮設計，客戶無需滾動頁面即可直接點擊付款；點數輸入區塊位於頁面內容區，容易在客戶未完整閱讀頁面的情況下被略過。

現有 Tiger Points 說明 bottom sheet 的三條內容：
1. TIGER POINTS 可用於折抵修繕訂單尾款。
2. TIGER POINTS 1 點折抵新台幣 1 元。
3. 本點數不可兌換現金或轉讓。

目前未說明「付款後無法補折抵」的限制，導致客戶對規則的預期與系統實際行為不符。

### 成功標準

客戶在支付尾款前能主動注意並確認是否使用點數；客服不再需要每日處理補折抵的退刷請求。

---

## 設計方向

本次優化包含兩個同步執行的改動：

### 支付尾款頁面 UI 調整

在置底懸浮區塊（付款按鈕上方）新增 Tiger Points 點數資訊提示，與付款按鈕同層，確保客戶在付款前必然看到點數相關資訊。顯示格式為「TIGER POINTS 折抵｜可用 X 點 >」，帶有箭頭表示可點擊互動。

**置底區塊的兩種狀態：**

| 狀態 | 點數資訊列顯示 | 付款按鈕顯示 |
|------|--------------|------------|
| 輸入點數為 0 | 「TIGER POINTS 折抵｜可用 N 點 >」（預設色，N = 帳戶可用點數） | 「前往付款 $A」（原始尾款金額） |
| 輸入點數 > 0 | 「TIGER POINTS 折抵｜使用 M 點折抵 >」（紅字，M = 已輸入折抵點數） | 「前往付款 $B」（折抵後金額，即時更新，B = A − M） |

點數實際扣除時機為**付款成功當下**，輸入點數僅代表意圖，尚未生效。

**互動方式：**
設計稿原方案為點擊後開啟獨立 bottom sheet 供輸入點數。工程師建議改為點擊後捲動頁面至點數輸入欄位，操作維持在既有頁面表單內，無需額外製作 bottom sheet。此方向已被接受，並需在捲動完成後自動 focus 點數輸入欄位，確保體驗流暢。

### Tiger Points 說明 bottom sheet 文案補充

在現有三條說明後新增第四條，明確告知客戶付款後無法補折抵：

> 請於支付尾款時確認是否折抵，**付款後恕無法補折抵。**

**視覺規格：**
- 後半句「付款後恕無法補折抵」使用粗體
- 無需紅字（此條為規則說明，無具體財務代價，不需警示層級）
- 排列於第 4 條（與第 3 條同屬限制類）

此 bottom sheet 有兩個入口：支付尾款頁的「說明」按鈕、客戶帳戶頁查看點數的入口，兩處共用同一 bottom sheet，文案更新後兩處同步生效。

## 決策記錄

**支付頁互動方式**
決定：點擊置底區塊的點數資訊後，捲動頁面至點數輸入欄位並自動 focus，不另開 bottom sheet。
為什麼：操作維持在既有頁面表單內，開發成本較低，體驗也足夠流暢。
放棄的替代方案：點擊後開啟獨立 bottom sheet 輸入點數（設計稿 v1 原方案）。

**置底區塊已輸入狀態的文案**
決定：「使用 M 點折抵 >」，紅字顯示。
為什麼：「使用」在中文不帶已完成語意，準確反映「付款成功才扣除」的時機；「折抵」與標題重複可接受，因為兩個狀態的結構差異（「可用」vs「使用⋯折抵」）搭配顏色區別，能提供足夠的視覺辨識度。
放棄的替代方案：「已折抵 M 點」（「已」暗示已扣除，與付款時機不符）、「折抵 M 點」（過於精簡，視覺區別不足）。

**是否在 bottom sheet 使用紅字**
決定：不使用紅字，僅粗體。
為什麼：此條為規則說明性質，無具體財務後果；紅字語氣偏重，且對帳戶內無點數的客戶干擾感強。董事長提議使用紅色標註，設計師評估後認為真正的解法在於 UI 層的可見度提升。
放棄的替代方案：全句紅字、前半粗體後半紅字。

**粗體範圍**
決定：僅後半句「付款後恕無法補折抵」加粗。
為什麼：前半是提醒行動，後半才是規則核心，粗體落在真正重要的地方。

**新條文排列位置**
決定：置於第 4 條（最後）。
為什麼：現有三條順序為「能做什麼 → 怎麼算 → 不能做什麼」，新條文同屬限制類，接續第 3 條語意最連貫。
