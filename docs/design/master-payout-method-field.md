# 師傅詳情頁新增「請款方式」欄位

## 概述

- **類型**：既有功能優化（新增欄位）
- **受影響角色**：客服（會計）、師傅

---

## 問題定義

### 痛點

對帳作業師傅視角需要顯示每位師傅的請款方式，但資料庫目前無此欄位，導致會計無法在系統內直接識別師傅的請款類別，影響撥款作業效率。

### 現況

- 師傅的請款方式未存入系統，無欄位記錄
- 對帳人員需依賴人工判斷或外部紀錄確認每位師傅適用哪種請款類別

### 成功標準

- 後台師傅詳情頁可顯示並編輯每位師傅的請款方式
- 對帳作業師傅視角可讀取並顯示此欄位
- 既有師傅資料補填前，系統可正確呈現「未設定」狀態

---

## 設計方向

### UI 元件：Radio Group

採用 Radio Group，三個選項橫排顯示於「媒合距離」與「銀行帳號」之間（以分隔線區隔上下語意群組）。

選用 Radio Group 而非 Dropdown 的理由：選項固定為三個，Radio 讓三個選項在檢視與編輯模式下皆直接可見，便於客服逐一補填既有師傅資料時快速判斷狀態。

### 欄位名稱與選項

| 項目 | 值 |
|------|---|
| 欄位標籤 | 請款方式 |
| 選項一 | 開立發票 |
| 選項二 | 整筆個人 |
| 選項三 | 分期個人 |

選項名稱對應師傅公告（[請款類別公告](https://blog.fdtigermaster.com/payout-request-policy/)）用語，後台採簡短版本。

### 檢視 / 編輯模式行為

| 模式 | 行為 |
|------|------|
| 檢視模式 | Radio disabled，選中項以藍色圓點標示；未設定則三個選項全部未選中 |
| 編輯模式 | Radio 可點選，三選一；後台客服可隨時修改 |

### 空值規格

- 資料庫欄位允許 `null`，代表尚未設定
- `null` → 前端三個 Radio 全部未選中，**不預設選中任何一個選項**
- 不新增「尚未設定」Radio 選項，避免客服主動選回空值清除已填資料
- 對帳作業師傅視角讀取到 `null` 時，顯示文字「未設定」而非空白

### 修改權限

- **後台**：客服可隨時修改
- **師傅端**：請款方式一經加入時決定後不可自行更改（非本次後台開發範圍）

---

## 影響範圍

- **師傅詳情頁**（後台）：師傅專用區塊新增「請款方式」Radio Group
- **對帳作業師傅視角**：讀取並顯示「請款方式」欄位，`null` 顯示「未設定」
- **資料庫**：師傅資料表新增請款方式欄位，允許 `null`

## 決策記錄

**採用 Radio Group 而非 Dropdown**
→ 選項只有三個，Radio 讓選項直接可見，適合逐筆補填作業；Dropdown 需點開才能看到選項，且空值狀態（「請選擇」）視覺語意不夠清楚
→ 放棄方案：Dropdown

**不新增「尚未設定」Radio 選項**
→ Radio Group 全部未選中本身即代表 `null`，額外加選項會製造客服誤操作風險（可能主動選回空值）
→ 放棄方案：加入第四個「尚未設定」選項

**不預設選中任何選項**
→ 預設值會讓既有未設定資料看起來像已設定，對帳時無法區分真實設定與系統預設，增加錯誤風險
