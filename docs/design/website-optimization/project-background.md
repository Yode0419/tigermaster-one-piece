# 官網優化專案 — 背景與資料夾說明

_這份文件記錄本次官網優化專案的背景，以及資料夾內各檔案的用途，方便回顧與後續協作。設計討論過程透過 `/luffy` 進行，正式收斂後會另外用 `/write-doc` 產出功能規劃文件。_

## 專案背景

董事長透過 Line 指派本專案，希望調整公司官網（師虎來了）首頁與關於我們頁面的內容。起因是近期陸續有人反映，師虎來了在商業模式上的差異化——相較於「找師傅」「PRO360達人網」等電話簿型平台，師虎來了涉及完整的案件管理與服務保障——並沒有在官網上清楚傳達出來。

董事長提供了一份簡報，列出他想調整的方向，並表示「大致方向，不一定要照著做」。

**改版基準**：以 PPT 內容為骨架，在既有官網內容上擴充＋微調，不需大幅翻新既有頁面結構。

## 資料夾內容說明

- **`官網調整1150716.pptx`** — 董事長提供的原始簡報，列出他想調整的方向與痛點（品牌差異化四項保障、企業福委會報名機制、關於我們頁的創辦人的話／陪伴感敘事／AI 願景、里程碑時間軸更新至 2028）。
- **`homepage-redesign-strategy-playbook.md`** — 使用者在其他專案做 landing page 改版時整理出的流程方法論（前期研究 → 策略定案 → 內容生產 → 原型製作 → 評審定案 → 工程交接，決策留存貫穿全程），作為這次專案可參考的流程框架，目前為空白模板，尚未套用到本專案內容。
- **`官網首頁與關於頁面設計/`** — 使用者用 Claude Design + Fable5 針對首頁與關於我們頁面所做的介面設計提案（design canvas 匯出檔），**尚未給董事長確認過**，目前是探索性版本：
  - `官網改版.dc.html` — 涵蓋首頁＋關於我們頁，其中「商業模式差異化」「企業福委會」「品牌宣言」等區塊用 Claude Design 的 tweaks 功能做了多版本文案／風格變體，可切換比較。
  - `首頁正式版.dc.html` — 首頁的一個整合定版（13 個區塊）。
  - `_ds/` — 設計提案沿用的設計系統 tokens（色彩、字體、效果）。
  - `assets/` — 提案使用的素材圖片（banner、插圖、服務項目圖、社群／商店圖示等）。
  - `uploads/` — 原始上傳素材，含 PPT 副本與 pptx 轉出的媒體圖片。

## 目前已知的關鍵資訊（設計討論已確認事項）

- **目標讀者優先順序**：一般消費者（客戶）＞ 企業福委會窗口 ＞（推測，非董事長明講）投資人 ＞ 師傅招募（現況維持不動，招募橫幅不改）。
- **差異化訴求對照對象**：「找師傅」「PRO360達人網」等電話簿型平台，差異點在於案件管理與服務保障，這件事目前官網文案與結構都沒有清楚呈現。
- **企業福委會報名機制**：確定採用既有 [Google 表單](https://docs.google.com/forms/d/e/1FAIpQLSfiSAoRjo2t17BMcRZ20uDt_BuNgBsPHjBonmr6CIE_29rjpQ/viewform) 連結做 CTA 導流，不採用 dc.html 提案中的客製化嵌入表單（避免資料來源分裂）。
- **成功標準**：一般消費者看完能理解師虎來了與其他平台的服務模式差異（甚至理解為何收費較高）；企業福委會窗口看完會被吸引並完成報名／聯繫。
- **現況**：既有官網（Vue.js，`Home.vue` / `About.vue`）已有基本架構，缺的是「案件管理差異化」這件事的清楚傳達，屬於內容＋結構都需要補的狀況。
