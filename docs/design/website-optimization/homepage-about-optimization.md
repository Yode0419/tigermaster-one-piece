# 官網首頁與關於我們頁內容優化

## 概述

- **類型**：既有功能優化
- **受影響角色**：客戶（一般消費者，官網主要訪客）、企業福委會窗口（B2B 對象）；師傅招募橫幅維持現況不動

---

## 問題定義

### 痛點

相較「找師傅」「PRO360達人網」等電話簿型平台，師虎來了在案件管理與服務保障上的差異化，目前官網內容與結構都沒有清楚傳達。消費者感受不到「為什麼比較貴」的理由；企業福委會目前沒有明確的報名機制銜接到官網上。

### 現況

現有官網（Vue.js）首頁（`Home.vue`）已有：頂部師傅招募橫幅、輪播、HOW IT WORK 五大優勢、大圖、五步驟詳解、APP 下載、八大服務項目、八步驟流程、客戶案例、合作夥伴，共 10 個區塊。關於我們頁（`About.vue`）已有：Banner、企業簡介、核心人物（陳洛婕）、核心價值三大支柱、品牌故事、未來展望，共 6 個區塊。現況文案已逐字整理於同資料夾內的 `current-homepage-content.md` / `current-about-content.md`。整體能傳達「透過平台找師傅很方便、有保障」，但沒有明確凸顯「案件管理」這件事的差異化。

### 成功標準

一般消費者看完首頁，能理解師虎來了與其他平台服務模式的差異，甚至理解為何收費較高；企業福委會窗口看完，能被吸引並完成報名/聯繫。

## 設計方向

以董事長 PPT 內容為骨架，在既有官網頁面上擴充+微調，不做大幅結構翻新。已有一份 Claude Design + Fable5 的探索性設計提案（`docs/design/website-optimization/官網首頁與關於頁面設計/` 資料夾，尚未給董事長確認過），`首頁正式版.dc.html` 中部分區塊做了 tweaks 多版本切換（`sc-if`），討論結果如下：

- **商業模式差異化區塊**：採用 `bizNumbers` 版本（大數字+四張保障卡+「師虎 vs 自己找師傅」對照表，已是檔案預設版本）。理由：對照表是直接回應「理解服務模式差異」的工具，數字開場呼應「憑什麼比較貴」的疑問。排除 `bizCard`（深藍主打卡）、`bizJourney`（5步驟旅程）——兩者皆無對照表；`bizPhotoTable`（照片對照表）可作備選。
- **企業福委會區塊**：採用 `welCards` 版本（三大權益卡+企業列表，已是檔案預設版本），CTA 按鈕改連到既有 [Google 表單](https://docs.google.com/forms/d/e/1FAIpQLSfiSAoRjo2t17BMcRZ20uDt_BuNgBsPHjBonmr6CIE_29rjpQ/viewform)，不採用 `welForm`（嵌入表單版），避免與 Google 表單資料來源分裂。理由：三大權益卡是唯一同時具備具體好處說明的版本，B2B 決策者需要先看到具體利益才會聯繫。原「合作夥伴」區塊移除，企業列表併入此區塊，移除台灣大哥大、工業技術研究院（非現況合作對象），新增重樂、育達科大等本次確定合作對象；入口位置除原規劃外也會嘗試加入 navbar 排版測試。
- **品牌宣言區塊**：採用 `stmtSplit`（左文右圖，已定案）。理由：白底左文右圖排版與官網其他區塊視覺調性較一致，資訊層級清楚（標題/內文/雙 CTA/圖片），不像 `stmtPhoto`（全幅照片帶）那樣視覺強烈到跳脫既有風格，也比 `stmtBlue`（淡藍宣言，內含三張保障小卡）更聚焦。
- 首頁宣言「找師傅，不該靠運氣」與關於我們頁宣言「修繕，不該是一個人孤軍奮戰」調性不同，經確認**不需要刻意統一**，可並存。

## SEO 角度分析（技術稽核，待後續詳細處理）

對 `fdtigermaster-offical-site` repo 做的技術稽核。總體結論：此站為 Vue 2.7 CSR SPA，但 build 時透過 `PrerenderSPAPlugin`（Puppeteer）將首頁、關於我們頁等路由預先渲染成靜態 HTML，爬蟲可讀性本身無重大問題；風險集中在 meta 資訊與標題結構這兩塊，剛好都能搭這次內容 revamp 一併處理。

### 技術 SEO

- 全站共用同一份 `<title>` / `<meta name="description">`（`public/index.html`），無任何 per-route 機制，首頁與關於我們頁預渲染後標題描述完全相同
- `<html lang="">` 為空字串，缺少 `lang="zh-TW"`
- 無 JSON-LD 結構化資料（缺 Organization / LocalBusiness schema）
- `public/sitemap.xml` 過時（`lastmod` 全為 2022-05-20），`/JoinUs` 重複列出兩次
- OG 標籤存在但全站共用一份；無 Twitter Card 標籤

### 內容結構 SEO

- **Home.vue 完全沒有 `<h1>` 或 `<h2>`，所有標題皆為 `<h3>`**，首頁沒有任何 h1 宣告頁面主體 —— 目前發現的最明確結構缺陷
- About.vue 有唯一 `<h1>`，階層正確，但手機版/桌機版各維護一份重複 DOM，「核心價值」「核心人物」等 `<h2>` 頁內重複出現
- `Home.vue:287` 服務項目圖片在 v-for 迴圈中 8 張圖全寫死同一個 `alt="service_01"`，未使用實際服務名稱；多處合作夥伴 logo 也共用同一組 alt 文字
- 兩頁幾乎全用 `<div>` + Bootstrap Vue，未使用語意化標籤（`<section>`/`<article>`）

### SEO 待辦（後續詳細處理）

> 補充脈絡：目前用品牌詞「師虎來了」搜尋 Google 已排第一、AI 摘要也能正確描述，但這是品牌詞查詢的預期結果——沒有競爭者會搶別人專屬的品牌詞，不代表網站 SEO 體質良好。以下待辦真正要解決的是**非品牌詞**（如「水電師傅推薦」「抓漏 桃園」等使用者還不知道品牌名時的服務型/在地型搜尋），這類查詢才有真實競爭、也才是能帶來新客的流量。目前的技術缺口不影響現有品牌詞搜尋體驗，屬於「未來要靠 SEO 拿非品牌新客流量前，需要先補的地基」，非緊急問題。

**預期效果**（技術地基工程，非直接拉排名——排名仍取決於內容深度與外部連結）：

- 非品牌詞搜尋結果的點擊率提升（title/description 不再跨頁重複、更貼合查詢意圖）

## 決策記錄

1. **企業福委會報名機制** → 決定連到既有 Google 表單（CTA 導流）→ 為什麼：避免跟嵌入表單產生兩份資料來源，且福委會窗口的成功標準是「被吸引報名」，表單精緻度非關鍵 → 放棄：dc.html 提案中的 `welForm` 嵌入式表單版本
2. **商業模式差異化區塊** → 選定 `bizNumbers`（已是預設）→ 為什麼：對照表是回應「理解差異」最直接的工具 → 放棄：`bizCard`、`bizJourney`（皆無對照表）
3. **首頁/關於我們頁宣言語氣** → 決定不強制統一 → 為什麼：兩頁可各自對應不同訴求（首頁重差異化/信賴、關於我們頁重陪伴感）→ 放棄：統一成單一調性的方案
4. **五大優勢區塊去留** → 決定保留（可壓縮但不刪除）→ 為什麼：總經理認為五大優勢是給客人看的產品優勢介紹，多數消費者不會認真逐項閱讀比較表，兩者功能互補、不互斥 → 放棄：只留比較表、刪除五大優勢的版本
5. **比較表比較項目與文案調性** → 決定為四項比較：付款方式（凸顯刷卡付款的安全性）、交易安全、保固、找師傅／媒合師傅的速度；文案要清楚易懂 → 為什麼：清楚易懂比刻意模糊化更能達成「讓消費者理解差異」的目標；同時避免直接使用競品的特定行銷用詞（如「安心支付」），降低被誤認為針對性攻擊、引發糾紛的風險 → 放棄：刻意寫得模糊、不明講比較項目的版本
6. **商業模式差異化五大重點的視覺焦點數字** → 決定用「訂金:尾款 = 3:7」付款比例作為最吸睛的數字重點 → 為什麼：具體數字比文字描述更快讓消費者抓到重點，且直接呼應付款安全訴求
7. **品牌宣言區塊的兩顆 CTA 按鈕** → 決定一顆捲動至商業模式差異化區塊（「看看師傅哪裡不一樣」）、一顆導向 App 下載 → 為什麼：手機版頂部已有獨立的 App 下載橫幅，品牌宣言區塊可以先引導使用者看完差異化內容再決定是否下載，避免使用者太快跳出頁面
8. **客戶案例區塊** → 決定新增「客戶評價／真實留言」區塊，取代客戶案例現有定位 → 為什麼：客戶案例目前只是 WordPress 精選文章、缺乏實質內容；後台有大量真實客戶留言，董事長認為比案例更有說服力 → 放棄：維持現有客戶案例區塊不變
9. **里程碑三大指標數字的曝光位置與呈現方式** → 決定同時放在首頁（品牌宣言／客戶評價區塊附近）與關於我們頁（里程碑時間軸），且數字要有跳動增加（count-up）動畫效果 → 為什麼：總經理認為能讓進站客戶第一時間感受到平台規模與活躍度；董事長認為跳動動畫比靜態數字更吸引注意、更有活力感 → 放棄：只放在關於我們頁、數字靜態呈現不做動畫
10. **合作夥伴區塊去留** → 決定移除合作夥伴區塊，改由企業福委會區塊取代（企業福委會區塊內含合作/認可企業清單，取代原合作夥伴的呈現功能）→ 為什麼：合作夥伴清單已過時（部分企業如台灣大哥大、工業技術研究院已非現況合作對象）、且與企業福委會功能重疊，維持兩個獨立區塊沒有必要；企業福委會清單改列本次確定合作的重樂、育達科大、中華電信福委會（已確認授權，須標示為「中華電信福委會」而非中華電信本體）等，遠傳電信尚未確認授權暫不列入 → 放棄：合併但保留兩個區塊並存的版本
11. **企業福委會入口位置** → 決定除了原規劃的頁面內位置外，也嘗試將入口加入 navbar 排版測試（尚未定案是否採用）→ 為什麼：增加入口能見度，但需先確認排版是否會與現有導覽列項目衝突
12. **關於我們頁導覽入口** → 決定維持現有頂部導覽列入口，不額外新增 footer 入口 → 為什麼：頂部導覽已有明確入口，footer 再加一個功能重複、無必要
13. **品牌宣言最終版本** → 決定採用 `stmtSplit`（左文右圖）→ 為什麼：排版與官網既有風格較一致、資訊層級清楚，雙 CTA（下載 APP／看看師虎哪裡不一樣）保留 → 放棄：`stmtPhoto`（全幅照片帶，視覺過於強烈）、`stmtBlue`（淡藍宣言，內含三張保障小卡、資訊量偏多）
14. **投資人受眾定位** → 決定正式列為官網目標讀者之一 → 為什麼：里程碑數字等內容需同時兼顧投資人視角撰寫文案 → 放棄：不特別列入、或維持不確定狀態
15. **客戶評價／真實留言區塊實作細節** → 決定：來源為 App/平台內完工後的評價欄位；篩選標準為人工精選高分且內容完整的評價；呈現方式為自動慢速跑馬燈 → 為什麼：平台內建評價資料真實可信，人工精選確保品質，跑馬燈呼應會議中「陳小姐慢慢跑過去」的視覺想像 → 放棄：規則自動篩選、手動切換卡片輪播
16. **中華電信／遠傳電信授權狀態** → 中華電信已確認可使用，但需標示為「中華電信福委會」（非電信公司本體）；遠傳電信尚未確認，暫不列入 → 為什麼：需精確標示合作實體，避免誤植為公司整體背書
