# 對帳與撥款作業優化：主管討論結果與調整方向

_2026-04-14 與董事長、總經理討論，本文件記錄確認事項、新增規格與待定事項_

## 概述

- **類型**：既有功能優化（規格確認與補充）
- **受影響角色**：客服（會計）
- **相關文件**：[對帳作業優化：師傅視角與撥款操作改善](reconciliation-optimization.md)

---

## 已確認決策

### 1. 匯款資訊表格支援週期切換與下載

匯款資訊的表格資料改為可選擇週期，切換後表格內容對應更新，並可下載為 Excel。

下載的 Excel **不含**平台開給客戶的發票資料。

### 2. 新增收入類型：施工中間款項請領

收入類型除現有的完工收款外，新增「施工過程中間款項請領」。入口位置可能在訂單詳情中的某個操作功能（待確認），需同樣能觸發寫入師傅收入的機制。

### 3. 取消對帳需求

後台若能確保資料抓取正確，對帳步驟可以省略，作業焦點聚焦在撥款操作本身。

### 4. 毛利率問題暫緩處理

毛利率算錯與算法不一致的問題目前暫不處理，維持參考性質。會計需要關注的只有毛利率顯示為**負值**的資料，作為異常訊號。

### 5. 師傅資料欄位擴充（報稅用途）

師傅資料除請款方式外，需增加以下欄位，供報稅使用（現況只能從證照的備註欄查看）：

| 欄位 | 用途 |
|------|------|
| 身分證字號 | 報稅 |
| 統編 | 報稅（公司戶） |
| 地址 | 報稅 |

**請款方式修改政策**：不開放師傅透過 App 自行修改，需透過客服申請異動。
- **新師傅**：說明會加入時填寫
- **舊師傅**：隨撥款作業推進，有缺漏時逐步補齊

### 6. 分期個人請款金額固定

分期個人請款固定金額為 **$19,999**，不可調整。

### 7. 撥款流程維持兩段式，改以師傅為視角

確認現有設計方向（見 [reconciliation-optimization.md](reconciliation-optimization.md)）：

- 維持「加入匯款紀錄 → 確認匯款」的兩段式流程
- **前期未匯**與**本期要處理的收入**，改以師傅為視角呈現
- 操作流程：會計將前期未匯與本期收入加入待處理區（師傅視角）→ 至銀行執行匯款 → 回系統確認匯款 → 狀態更新為匯款紀錄

## 決策記錄

**不對帳，只撥款**
→ 後台資料正確的前提下，對帳步驟可省略
→ 放棄方向：強化系統對帳功能

**毛利率問題暫緩**
→ 現況僅供參考，正確性非當前要務；只需人工關注負值資料
→ 放棄方向：修正毛利率計算邏輯與算法一致性

**請款方式不開放 App 自改**
→ 請款方式屬財務關鍵設定，需透過客服流程審核才可異動

**分期請款金額固定 $19,999**
→ 每期固定金額，不允許自訂調整
