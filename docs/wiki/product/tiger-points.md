# Tiger Points

## 概述

Tiger Points 是 Pro 會員取得的點數福利，可於尾款支付時折抵訂單金額，1 點折抵新台幣 1 元。

---

## 背景與脈絡

Tiger Points 為升級 Pro 會員時一次性取得的點數貨幣，行銷包裝為「修繕金翻倍」的核心訴求之一。與免派遣費次數不同，客戶需主動在尾款支付頁面選擇使用。詳細的 Pro 會員方案說明請參考 [Pro 會員（師虎 Pro）](pro-membership.md)。

## 取得方式

| 來源 | 點數 | 說明 |
|------|------|------|
| 升級 Pro 會員 | 4,000 點 | 行銷包裝為「回饋 2,000 點＋再贈 2,000 點」，一次性取得 |
| 不定期活動 | 不定 | 例如吳淡如粉絲活動曾額外贈送 100 點，視活動設計而定 |

## 查看入口（客戶端）

客戶可在兩個地方查看剩餘點數：

- **帳號頁面**：與免派遣費次數並列顯示，點入可查看詳情頁，包含當前餘額及使用說明。詳情頁目前不顯示歷史折抵紀錄
- **尾款支付頁**：有剩餘點數時顯示 Tiger Points 折抵區塊，內含當前餘額；餘額為 0 時不顯示此區塊

## 使用流程

尾款支付流程分為兩個頁面：

**Page 1 — 已完成驗收**

顯示完工資訊（工項、完工時間、保固期間），底部 CTA 為「下一步：確認付款資訊」。此頁不涉及付款操作。

**Page 2 — 確認付款資訊**

- 頁面頂部顯示「TIGER POINTS 折抵」區塊，列出當前餘額
- 客戶手動輸入欲折抵的點數，或點擊「全部折抵」一鍵帶入
- 折抵金額即時反映於下方支付明細
- 頁面底部提示：「付款後恕無法補折抵」
- 底部 CTA 顯示折扣後實付金額

> **設計背景**：早期版本為單頁付款，Tiger Points 折抵欄位容易被客戶略過，曾引發客戶未使用點數的客訴。拆為兩頁後，折抵區塊固定呈現於 Page 2 頂部，確保客戶不會錯過。

## 驗證規則

| 情境 | 錯誤提示 |
|------|---------|
| 輸入點數超過訂單應付金額 | 折抵不能超過應付金額 |
| 輸入點數超過帳戶可用餘額 | 超過可用點數上限 |

## 適用時機

- 升級時訂單**尚未報價** → 可於當前訂單尾款折抵
- 升級時訂單**已完成報價** → 該訂單不適用，從下一張訂單起生效

## 其他限制

- 不可兌換現金，不可轉讓
- 無使用期限，但 Pro 會員資格終止即失效

## 後台管理能力

客戶詳情頁的「點數」tab 顯示當前剩餘點數與完整的點數異動紀錄：

| 欄位 | 說明 |
|------|------|
| 項目 | 異動類型，如「手動更改」、「尾款使用點數折抵」 |
| 訂單 id | 關聯訂單（可點擊跳轉）；手動更改無訂單 |
| 點數增減 | 正值（+）為增加，負值（−）為扣除 |
| 剩餘點數 | 異動後的餘額 |
| 註記 | 手動更改時可填寫的備注文字 |
| 建立者 | 操作者帳號 id |
| 建立日期 | 異動時間 |

右上角「更改點數」按鈕可手動增減，調整時可填寫備注。
