# 完工後評價與小費機制

## 概述

本文件描述訂單結案後，客戶與師傅互相評價的機制，以及客戶給予小費的功能。評價資料僅供後台紀錄與師傅管理使用，不對外公開。

---

## 相關角色

| 角色 | 說明 |
|------|------|
| 客戶 | 驗收並完成尾款支付後進行評價，可選擇給小費 |
| 師傅 | 客戶同意驗收後可對客戶進行評價 |
| 客服管理師 | 可能在案件過程中擔任溝通橋樑或監工，客戶可選擇給小費 |

---

## 評價機制

### 師傅評客戶

- **觸發時機**：客戶同意驗收後
- **評分方式**：星數（最多 5 顆）＋文字評論
- **可見範圍**：僅記錄於後台，不對外公開

### 客戶評師傅

- **觸發時機**：驗收同意且完成尾款支付後（若無尾款則驗收後即可）
- **評分維度**：施工品質、時間掌握、服務態度（各最多 5 顆星）
- **可見範圍**：評論僅記錄於後台，不對外公開

### 評價是選填的

App 在完工後一次性顯示評價表單畫面，關掉後不再出現，平台不強制要求填寫。訂單狀態在尾款支付後進入「等待完成」，不因評價而阻擋結案。

---

## 小費功能

客戶完成評價後，可選擇對師傅或客服管理師給予小費。

| 項目 | 說明 |
|------|------|
| 給小費對象 | 師傅、客服管理師，各自獨立輸入 |
| 金額選項 | 預設 $100 / $200 / $300 / $500，亦可自由輸入 |
| 支付方式 | 僅限刷卡 |

客服管理師收到小費的情境，通常是其在案件過程中扮演溝通橋樑、或親自到場協助監工與驗收，客戶認為服務貼心而主動給予。

**分潤規則**：師傅實際獲得客戶給付小費金額的 **70%**，平台抽成 30%（平台解釋為營運費用）。此規則曾有師傅反映不合理、認為應領全額，平台維持原規則未調整。

後台提供「**訂單小費**」功能區塊，可查看所有有支付小費的訂單及金額明細。

---

## 師傅星數顯示

系統依師傅歷次訂單的客戶評分計算平均星數，並在媒合成功後、客戶等待支付派遣費的畫面上顯示該師傅的評分。

**新師傅預設 5 顆星**：尚無評分紀錄的師傅，系統預設顯示 5 顆星。設計動機為提升客戶信任感；但因客戶無法自行挑選師傅，此顯示的實際參考價值有限。目前無改版規劃（如改顯示「尚無評分」等）。

---

## 評價資料的實際用途

| 用途 | 說明 |
|------|------|
| 後台師傅管理 | 記錄每位師傅的歷次表現，供客服掌握師傅品質 |
| 媒合參考 | 客服主動尋找師傅時，優先挑選有接案紀錄且表現正常者；師傅不足時才開發新師傅 |
| 對客戶的參考價值 | 有限——客戶無法挑選師傅，僅在等待支付派遣費畫面上被動看到星數 |
