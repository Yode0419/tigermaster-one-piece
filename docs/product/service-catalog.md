# 服務工項架構

## 概述

師虎來了平台的工項系統由兩套獨立結構組成：面向客戶的三層分類導航（L1-L2-L3），以及面向師傅媒合的技能系統。兩者透過工項編號前綴相互對應，但設計目的與運作邏輯完全分離。

---

## 客戶端導航：L1-L2-L3 分類

三層分類的唯一用途是幫助客戶找到對的工項來下訂單，不涉及師傅媒合邏輯。

### L1 大分類（共 8 類）

| 編號 | 名稱 |
|------|------|
| 1 | 安裝服務 |
| 2 | 水電服務 |
| 3 | 清潔服務 |
| 4 | 裝潢整修 |
| 5 | 搬運服務 |
| 6 | 家電服務 |
| 7 | 房屋漏水 |
| 8 | 消毒病媒 |

L1 之下依序細分為 L2 中分類，再細分至 L3 工項。目前平台共有約 200 個工項。

### L3 工項欄位說明

每個 L3 工項包含以下欄位，展示於官網與 App 供客戶下訂單時參考：

| 欄位 | 說明 |
|------|------|
| 工項編號 | 唯一識別碼，格式見下方說明 |
| 工項名稱 | 顯示於前端的工項名稱 |
| 工項描述 | 說明常見情境，協助客戶確認是否為所需服務 |
| 最高價錢 | 報價參考上限（不具報價限制效力） |
| 最低價錢 | 報價參考下限（不具報價限制效力） |
| 常見價格 | 多數案件的典型落點（參考用） |
| 價格區間說明 | 補充說明影響報價的因素 |
| 普通使用保固 | 一般住宅使用的保固天數 |
| 商業使用保固 | 商業場所使用的保固天數（部分工項無保固） |
| 保固描述 | 說明保固責任範圍與處理方式 |
| 啟用 | 控制該工項是否顯示於前端 |

**價格說明**：所有價格欄位均為參考資訊，由總經理與客服依實務案件經驗設定，非統計數據，可隨時透過後台調整。實際報價由總經理人工審核報價內容合理性，最高、最低價格欄位不構成任何報價限制。

**保固說明**：有保固的工項，保固責任由師虎來了平台承擔。在保固期間內，若因施工不良或材料不當造成損害，並經雙方確認後，師虎來了將聯繫原師傅負責維修處理（平台本身無內部維修人員）。無保固的工項通常為「安裝服務」類，或視工項性質而定（例如修繕顧問類工項）。

### 範例工項：洗衣機故障（TM-A020101）

| 欄位 | 內容 |
|------|------|
| 工項編號 | TM-A020101 |
| 工項名稱 | 洗衣機故障 |
| 工項描述 | 可下單前備註常見的故障原因，如：全不動作、無法脫水或脫水時有撞擊聲、排水不暢、不進水、脫水速度很慢或衣服脫不乾等等。 |
| 最高價錢 | 12,000 |
| 最低價錢 | 1,800 |
| 常見價格 | 3,500 |
| 價格區間說明 | 依據不同問題報價將有所差異，基本維修出工費為 $1,800。 |
| 普通使用保固 | 31 天 |
| 商業使用保固 | 31 天 |
| 啟用 | 是 |

## 師傅媒合：技能系統

技能系統與 L1-L2-L3 完全無關。

**媒合邏輯**：系統核心是比對師傅是否具備客戶所選 L3 工項的資格。由於逐一將工項加諸師傅身上效率較低，因此以「技能」作為批次管理的單位——師傅開通一個技能，即同時取得該技能下所有工項的承接資格（也可排除特定工項）。

### 工項編號結構

```
TM - [類別字母] [技能序號] [工項序號]
```

- **末四碼為 `0000`**：代表技能本身（例：`TM-A020000` = 洗衣機技能）
- **末四碼非零**：代表該技能下的具體工項（例：`TM-A020101` = 洗衣機故障）

**範例：洗衣機技能（TM-A020000）**

| 工項編號 | 工項名稱 |
|----------|----------|
| TM-A020101 | 洗衣機故障 |
| TM-A020102 | 洗衣機清洗 |

---

## 後台管理

L1-L2-L3 分類與 L3 工項的新增、修改、刪除、啟用／停用，均透過後台的「技能」、「工項」、「L1-L3 對應」功能區塊進行。完整工項清單以後台資料為準。
